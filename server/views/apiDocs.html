Authentication Endpoints

Secure user registration and authentication system

/api/auth

User authentication and authorization endpoints

POST /api/auth/signup Public
Register a new user account in the Artosia marketplace. Creates a new user with encrypted password and returns user information. Both username and email must be unique.
Request Body
{
  "username": "string (required, unique)",
  "name": "string (required)",
  "mobile": "string (optional)",
  "email": "string (required, unique)",
  "password": "string (required)"
}
POST /api/auth/login Public
Authenticate user credentials and receive a JWT token for accessing protected endpoints. You can login using either username or email.
Request Body
{
  "username": "string (required - can be username or email)",
  "password": "string (required)"
}

Painting Management

Complete CRUD operations for artwork marketplace

/api/paintings

Manage paintings in the art marketplace

GET /api/paintings Public
Retrieve all paintings available in the marketplace. Returns a list of all artworks with complete details including artist and posting information.
Response Fields
Array of objects containing:
- title, imageUrl, price, paperType
- description, size, artistName
- category, postedBy (username)
POST /api/paintings Authentication Required
Add a new painting to the marketplace. Automatically associates the painting with the authenticated user (postedBy field is auto-filled with username).
Request Body
{
  "title": "string (required)",
  "imageUrl": "string (required)",
  "price": "number (required)",
  "paperType": "string (required)",
  "description": "string (required)",
  "size": "string (required)",
  "artistName": "string (required)",
  "category": "string (required)"
}
Auto-Generated Fields
postedBy: Automatically set to authenticated user's username
PUT /api/paintings/:id Owner Only
Update an existing painting. Only the original poster can modify their artwork details. The postedBy field cannot be changed.
URL Parameters
id: Painting ObjectId (MongoDB)
Request Body (Optional Fields)
{
  "title": "string",
  "imageUrl": "string",
  "price": "number",
  "paperType": "string",
  "description": "string",
  "size": "string",
  "artistName": "string",
  "category": "string"
}
DELETE /api/paintings/:id Owner Only
Permanently remove a painting from the marketplace. Only the original poster can delete their artwork.
URL Parameters
id: Painting ObjectId (MongoDB)

Authentication Guide

For protected endpoints marked with "Authentication Required":

# Include JWT token in Authorization header Authorization: Bearer <your-jwt-token> # Example using curl curl -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." \ -X GET http://localhost:5000/api/paintings

How to obtain a JWT token:

  1. Register a new account via /api/auth/register
  2. Login with your credentials via /api/auth/login
  3. Use the returned token in the Authorization header

Security Notes:

  • Tokens expire after 24 hours for security
  • Users can only modify their own paintings
  • All passwords are encrypted using bcrypt
